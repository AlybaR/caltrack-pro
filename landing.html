FAQ

Les bonnes questions

Si la réponse n'y est pas, écris-moi : lyno.app1@gmail.com

C'est gratuit, c'est où le piège ?
Pas de piège. Pas de pub, pas de paywall, pas de vente de données. Le projet est financé par mon temps libre. La sync Firebase a un free tier qui couvre largement les premiers milliers d'utilisateurs. Si l'app explose, j'ajouterai un don volontaire — jamais de paywall sur les fonctions de base.
C'est open-source ?
Oui, le code source est public sur GitHub : github.com/AlybaR/caltrack-pro. Tu peux fork, contribuer, ou héberger ta propre version.
Comment vous protégez mes données ?
Mode local-only par défaut : tes données ne quittent jamais ton appareil. Si tu crées un compte (optionnel) : chiffrement HTTPS, stockage Google Firebase serveurs UE, tu es le seul à y accéder. RGPD strict, droit d'effacement à tout moment. Politique complète →
Comparaison MyFitnessPal / Yazio / Lifesum ?
Lyno est plus simple, plus rapide, plus honnête. Pas de feed social, pas de coach virtuel qui culpabilise. Plus orienté privacy. Conçue pour des adultes français (saisons, marques, types de plats locaux). Idéale si tu cherches un tracker direct et neutre.
Disponible sur iPhone ?
L'app est une PWA (Progressive Web App), donc oui : sur iPhone, ouvre Safari → menu Partager → "Sur l'écran d'accueil". Tu obtiens une icône comme une vraie app. Une version native iOS App Store est prévue si l'app décolle.
Ça marche hors-ligne ?
Oui. C'est une PWA installable qui cache les ressources. Tu peux logger des aliments, du sport, ton poids même sans connexion. Le scan code-barres a besoin d'internet pour interroger OpenFoodFacts. La sync se fait au retour en ligne.
Avertissement médical ?
Lyno est un outil de tracking, pas un dispositif médical. Les calculs (BMR, TDEE, déficit, IMC) sont des estimations validées scientifiquement, mais ne remplacent pas l'avis d'un professionnel de santé. En cas de troubles alimentaires, antécédents médicaux, ou doute, consulte un médecin/diététicien.
